# CLAUDE.md - Project Rules for Claude Code

## Git Rules

- **mainブランチに直接commit/pushしない** — 作業は必ずブランチを作成して行うこと。mainへの直接変更は禁止。
- **commit/pushはユーザーの明示的な指示があるまで行わない** — コード変更の内容を提示し、承認を得てからcommit/pushする。勝手にcommitやpushをしない。
- **git addしたまま放置しない** — ステージングした変更は必ずcommitするか、不要ならunstageする。中途半端な状態で残さない。

## Build Artifacts

- **ビルドアーティファクトを変更しない** — `.open-next/`, `deploy/` ディレクトリはClaude側で編集・commit対象にしない。これらはビルドプロセスで自動生成されるもの。

## Work Style

- **試行錯誤のデバッグコミットを大量に作らない** — 動作確認は手元で行い、確定した変更のみをコミットする。`debug:` プレフィックスのコミットを連発しない。
- **変更の影響範囲が大きい場合は事前に方針を相談する** — 大規模なリファクタや新機能追加は、実装前にユーザーと方針を合わせる。
